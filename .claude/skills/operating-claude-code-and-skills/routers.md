# Routers: Intent → File

## Meta Rule (最優先)
- **任意のSkillファイルを編集する際** → **必ず最初に `operating-claude-code-and-skills/SKILL.md` と `workflows.md` を読む**

## Routing Rules
- **方針更新（原則の追加/変更）** → `reference.md`
- **使い方手順/チェックリスト** → `workflows.md`
- **短い要点の入口** → `SKILL.md`（Quick Rules / Checklist）
- **コード/Before-After** → `examples.md`
- **フォーマット雛形** → `templates.md`
- **評価シナリオ** → `evaluations.md`

## Examples
- 「Liquid Glass のコンテナ間距離の説明を詳しく」→ `reference.md`
- 「導入手順のチェックリストが欲しい」→ `workflows.md`
- 「要点をトップに1行追加」→ `SKILL.md`（箇条書き1つだけ）
- 「NG例に fixed color を追加」→ `examples.md`
